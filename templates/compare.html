📝 Algorithm Analysis Summary

Greedy (O(n log n))
Sorts items by value/price ratio and picks greedily. Optimal for fractional knapsack, but may miss the best solution for 0/1 knapsack when items can't be split.
Dynamic Programming (O(n×W))
Builds a 2D table of sub-problems. Always finds the globally optimal solution. Trade-off: uses O(n×W) memory and time, impractical for very large W.
Brute Force (O(2ⁿ))
Tries every possible subset — guaranteed optimal but exponentially slow. With 20 items that's 1,048,576 subsets. Demonstrates why algorithm design matters.
